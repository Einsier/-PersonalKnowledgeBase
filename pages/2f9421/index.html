Linux内核编译安装

# 安装步骤

操作环境：

  • Linux系统：Ubuntu 18.04
  • 原内核：5.4.0-91-generic
  • 新内核：5.4.22

# 查看本机内核版本

在终端内输入uname -r可查看本机内核版本

# 下载内核源码

可在https://mirrors.edge.kernel.org/pub/linux/kernel/ (opens new window)中下载各版本的内核源码，将其解压至目录/usr/src中：

$ sudo tar -zxvf linux-5.4.22.tar.gz -C /usr/src
1

# 安装必要软件

$ sudo apt install git fakeroot build-essential ncurses-dev xz-utils libssl-dev bc flex libelf-dev bison
1

# 内核配置

因为两个版本相近，拷贝与本机相同的配置文件：

$ cd /usr/src/linux-5.4.22  # 进入内核源码目录
$ sudo make mrproper  # 若此前编译过，使用该命令清理编译的残留文件（包括配置文件和中间文件）；否则跳过
$ sudo cp -v /boot/config-$(uname -r) .config  # 拷贝当前使用的配置文件至本目录下
1
2
3

使用命令sodu vim .config修改配置，图中所示参数置为空：

$ sudo make menuconfig  # 打开图形化配置界面，可进行修改操作；这里不做更改，直接保存退出
1

# 编译内核

在/usr/src/linux-5.4.22目录下执行：

$ sudo make clean  # 若此前编译过，清除中间文件（保留配置文件）
$ sudo make -j4 bzImage  # 编译内核，-j<number>：以多线程数并行编译
1
2

此过程较漫长，若编译成功，可在./arch/x86/boot/目录（根据终端信息可能有所不同）下找到bzImage

# 编译安装驱动

在/usr/src/linux-5.4.22目录下执行：

$ sudo make -j4 modules  # 构建所有驱动
$ sudo make -j4 modules_install # 安装驱动
1
2

此过程较漫长，若安装成功，可在/lib/modules/目录下新生成一个5.4.22目录

# 安装内核

在/usr/src/linux-5.4.22目录下执行：

$ sudo make -j4 install  # 安装新内核
1

若安装成功，查看/boot目录，将会生成initrd.img-5.4.22、System.map-5.4.22、vmlinuz-5.4.22三个文件，且/boot/grub/grub.cfg中也会更新新内核的相关信息

# 修改grub配置

执行命令sudo vim /etc/default/grub修改grub配置，修改图中所示参数：

执行以下命令更新grub配置使得启动时可选择内核版本：

$ sudo update-grub
1

# 验证结果

重启系统，选择新内核启动：

可使用命令uname -r查看系统内核版本是否已经更改

# 问题及解决

  1. 无法启动menuconfig：需要安装必要软件和依赖，过程参见步骤-安装必要软件
  2. 编译时报错No rule to make target 'debian/canonical-certs.pem', needed by 'certs/x509_certificate_list'：将.config中CONFIG_SYSTEM_TRUSTED_KEYS置空，过程参见步骤-内核配置
  3. 编译过程中磁盘空间不够，使用Gparted工具进行扩容，在此不做过多介绍
